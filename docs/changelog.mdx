<Update label="2025-09-14" description="v1.0.0-beta.1">
**New Features & Updates:**
- **Core:**
  - Initial release of inferedge-moss
  - Multi-index support for isolated search spaces
  - API key validation with secure host access
  - Performance metrics tracking
  
- **Search:**
  - Semantic search using transformer-based embeddings
  - Query interface with configurable result count
  - Add, update, and remove items across indexes

- **Embeddings:**
  - Lightweight embedding models for edge computing
  - Support for proprietary "moss-minilm" model
  - Cloudflare CDN support for fast model loading
</Update>

<Tab title="JavaScript">
<Update label="unreleased" description="v1.0.0-beta.3">
**New Features & Updates:**
- **Embeddings:**
  - Added support for user-supplied document embeddings during ingestion
  - Added optional `embedding` arrays in `DocumentInfo` payloads to bypass native embedding service
  
- **Search:**
  - Added `QueryOptions` overload to accept custom embeddings alongside query text
  
- **Core:**
  - Relaxed `modelId` requirement when creating indexes
  - SDK now defaults to `moss-minilm` model when no explicit model is provided
</Update>

<Update label="2025-11-30" description="v1.0.0-beta.2">
**Bug Fixes:**
- **Core:**
  - Fixed ESM (ES Module) import compatibility issue
  - Package now correctly exports as an ES module with standard ESM syntax support

**Breaking Changes:**
- **Core:**
  - Migration required from CommonJS (`require`) to ES Module syntax (`import`)
</Update>

<Update label="2025-10-01" description="v1.0.0-beta.1">
**New Features & Updates:**
- **Core:**
  - Initial release of @inferedge/moss
  - Multi-index support for isolated search spaces
  - TypeScript support with full type definitions
  
- **Search:**
  - Semantic search using transformer-based embeddings
  - Blazing fast querying support after loading indexes
  - Add, update, and remove documents across indexes

- **Embeddings:**
  - Lightweight embedding models for edge computing
  - Support for proprietary "moss-minilm" and "moss-mediumlm" models
</Update>
